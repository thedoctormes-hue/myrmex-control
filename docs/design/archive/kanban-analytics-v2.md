## 1. Обзор системы

### Принципы

- **Данные первичны.** Каждая метрика выводится из фактов (changelog + текущее состояние), а не из кэша или агрегатов.
- **Реальное время.** Аналитика пересчитывается при каждом запросе из актуального состояния myrmex.json.
- **Доказуемость.** За каждым числом стоит список задач, которые его формируют. Пользователь может "провалиться" от метрики к конкретным задачам.
- **Три уровня детализации:**
  - **Доска** — мини-виджеты прямо на канбан-странице (KPI-карточки, sparkline)
  - **Проект** — полные графики по одному проекту (CFD, cycle time, throughput)
  - **Колония** — сводная аналитика по всем доскам (сравнение ЗавЛаб vs МУРАВЕЙ vs КОТ)

### Архитектура

```
Frontend (React)                  Backend (Express)
┌──────────────────┐             ┌──────────────────────┐
│ Board Widgets    │             │ /api/analytics/*     │
│ Project Charts   │  REST API   │   CFD Calculator     │
│ Colony Overview  │ ◄─────────► │   Cycle Time Calc    │
└──────────────────┘             │   Throughput Calc    │
                                 │   WIP Calculator     │
                                 │   Aging Analyzer     │
                                 │   Agent Efficiency   │
                                 │   Health Score       │
                                 └──────────┬───────────┘
                                            │
                                       readState()
                                            │
                                       myrmex.json
```

## 2. Источники данных

### Первичные данные

| Источник | Поля | Использование |
|----------|------|---------------|
| Task[] | id, project_id, status, priority, assignee_id, created_at, updated_at, completed_at | Текущее состояние, WIP, распределение |
| ChangelogEntry[] | id, timestamp, action, entity_type, entity_id, diff | История переходов, CFD, cycle time, throughput |
| Agent[] | id, name, status, project_id, current_task_id | Загрузка агентов, эффективность |
| Project[] | id, name, status | Группировка по доскам |

### Вычисляемые поля

| Поле | Формула | Описание |
|------|---------|----------|
| cycle_time_hours | completed_at - created_at (часы) | Полное время жизни задачи |
| lead_time_hours | completed_at - (переход из backlog) | Время от принятия до завершения |
| time_in_status_hours | updated_at - (последний переход в статус) | Текущее время в статусе |
| age_hours | now - created_at | Возраст задачи |
| is_stale | time_in_status_hours > threshold | Задача застряла |

### Рекомендация по changelog

Для точного расчёта CFD и cycle time при каждом изменении статуса записывать структурированный diff:

```json
{
  "field": "status",
  "from": "todo",
  "to": "in_progress"
}
```

## 3. Кумулятивная диаграмма потока (CFD)

### Что это

CFD — основной инструмент анализа потока в Kanban. Показывает, как задачи накапливаются в каждом статусе во времени.

### Визуализация (stacked area chart)

- **X:** время (дни/недели, диапазон 7/14/30/90 дней)
- **Y:** кумулятивное количество задач
- **Слои:** статусы снизу вверх: backlog, todo, in_progress, review, done

### Цвета статусов

| Статус | HEX | Смысл |
|--------|-----|-------|
| backlog | #6b7280 | Не запланировано |
| todo | #3b82f6 | Запланировано |
| in_progress | #f59e0b | В работе |
| review | #8b5cf6 | На проверке |
| done | #22c55e | Завершено |
| cancelled | #ef4444 | Отменено |

### Интерпретация паттернов

| Паттерн | Интерпретация | Действие |
|---------|---------------|----------|
| Параллельные линии | Стабильный поток | Всё хорошо |
| Расхождение (веер) | Накопление в статусе | Узкое место |
| Схождение | Задачи пропадают | Проверить cancelled |
| in_progress расширяется | WIP растёт | Остановить набор задач |
| done выпрямляется | Пропускная способность упала | Разобраться с блокерами |

### Алгоритм

1. Определить диапазон [from, to]
2. Для каждого дня восстановить состояние всех задач через changelog
3. Посчитать количество задач в каждом статусе
4. Построить stacked area chart

### Метрики CFD

| Метрика | Описание |
|---------|----------|
| band_widths | Ширина полосы по дням |
| bottleneck | Статус с максимальным ростом |
| stability_index | 1 - (std_dev / mean) ширины in_progress |

## 4. Время цикла (Cycle Time)

### Определения

| Тип | Начало | Конец | Что показывает |
|-----|--------|-------|----------------|
| Lead Time | created_at | completed_at | Ожидание + работа |
| Cycle Time | Переход в in_progress | completed_at | Чистое время работы |
| Review Time | Переход в review | completed_at | Время на проверке |

### Метрики

| Метрика | Описание |
|---------|----------|
| mean | Среднее время цикла |
| median | Медиана (устойчива к выбросам) |
| p90 | 90% задач завершаются за это время |
| p95 | 95% задач |
| std_dev | Разброс (предсказуемость) |
| trend | Линейная регрессия по неделям |

### Разбивка по приоритетам

| Приоритет | Ожидаемый cycle time |
|-----------|---------------------|
| critical | < 4 часов |
| high | < 24 часов |
| medium | < 72 часов |
| low | < 1 недели |

### Визуализация

- **Гистограмма** распределения времени цикла (бины: 0-4ч, 4-8ч, 8-16ч, 16-24ч, 24-48ч, 48+ч)
- **Линия тренда** по неделям (mean/median)
- **Box plot** для сравнения по приоритетам

## 5. Пропускная способность (Throughput)

### Что это

Количество задач, завершённых за единицу времени. Мера продуктивности колонии.

### Метрики

| Метрика | Описание |
|---------|----------|
| daily_avg | Среднее завершённых задач в день |
| weekly_avg | Среднее завершённых задач в неделю |
| trend | Растущий / стабильный / падающий |
| variability | CV = std_dev / mean (< 0.3 = стабильный) |

### Прогноз

- **Бэклог готов через:** backlog_count / throughput.week недель
- **N задач завершены через:** N / throughput.day дней

### Визуализация

- **Bar chart** по дням/неделям
- **Сравнительный bar chart** по проектам (ЗавЛаб vs МУРАВЕЙ vs КОТ)

## 6. WIP-анализ

### Что это

WIP (Work In Progress) — задачи в статусах in_progress + review. Контроль WIP — сердце Kanban.

### Правила лимитов

| Состояние | Условие | Действие |
|-----------|---------|----------|
| Норма | WIP < 80% лимита | Можно брать задачи |
| Внимание | WIP 80-100% | Предупреждение |
| Превышение | WIP > лимит | Блокировка + алерт |

### Метрики

| Метрика | Описание |
|---------|----------|
| current | Текущий WIP |
| limit | Суммарный лимит |
| utilization | current / limit * 100% |
| avg_duration | Среднее время в WIP |
| violations_count | Дней с превышением за период |

### Закон Литтла

```
Cycle Time = WIP / Throughput
```

При WIP=12, Throughput=6/день → Cycle Time = 2 дня.

### Визуализация

- **Прогресс-бар** текущий WIP / лимит
- **Line chart** тренда WIP с линией лимита
- **Таблица** по колонкам: текущий / лимит / статус

## 7. Старение задач

### Что это

Задачи, слишком долго находящиеся в одном статусе — индикатор проблем.

### Пороги старения

| Статус | Жёлтый | Красный |
|--------|--------|---------|
| backlog | > 30 дней | > 60 дней |
| todo | > 14 дней | > 30 дней |
| in_progress | > 5 дней | > 10 дней |
| review | > 3 дня | > 7 дней |

### Алерты

| Условие | Уровень | Действие |
|---------|---------|----------|
| Задача > красного порога | Critical | Уведомление + подсветка |
| Задача > жёлтого порога | Warning | Подсветка на доске |
| 3+ задачи in_progress > порога | Critical | Остановить набор задач |
| WIP > лимит 3+ дня | Critical | Увеличить лимит или ускорить |

### Визуализация

- **Heatmap:** задачи по времени в колонке (строки = задачи, колонки = дни, цвет = возраст)
- **Таблица** стареющих задач: название, статус, дней в статусе, приоритет, агент
- **Индикаторы** на карточках задач (жёлтая/красная рамка)
