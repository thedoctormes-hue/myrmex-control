## 1. Обзор интеграции

### 1.1. Что уже есть

TWA-слой частично реализован в `src/client/shared/lib/twa.ts`:
- `initTWA()` — инициализация WebApp, расширение viewport
- `haptic()` — обёртка над HapticFeedback API
- `isTWA()` — определение запуска внутри Telegram
- `getTWAUser()` — данные пользователя из `window.Telegram.WebApp.initDataUnsafe`
- `getTWATheme()` — тема Telegram (цвета для кастомизации UI)
- MainButton / BackButton — базовые обёртки

### 1.2. Что нужно добавить

| Компонент | Статус | Приоритет |
|-----------|--------|-----------|
| MainButton на странице Board | Не используется | P0 |
| BackButton на вложенных экранах | Не используется | P0 |
| Deep link на карточки задач | Нет | P0 |
| Создание задач из TG-сообщений | Нет | P0 |
| Inline keyboard для быстрых действий | Нет | P1 |
| Уведомления через TG-ботов | Частично (см. automation) | P1 |
| Голосовой ввод задач | Нет | P2 |
| Шеринг карточки в TG-чат | Нет | P2 |

### 1.3. Принцип: TWA — расширение, а не зависимость

Канбан-система должна работать как в TWA, так и в обычном браузере. Все TWA-фичи — прогрессивное улучшение:
- `isTWA()` → включаем MainButton, BackButton, haptic
- `!isTWA()` → используем обычные кнопки навигации, без haptic
- Никаких блокирующих зависимостей от Telegram API

## 3. Навигация в TWA

### 3.1. Точки входа

Пользователь попадает в TWA через:

1. **Меню-кнопка бота** (Menu Button) — постоянная кнопка в поле ввода чата с ботом
2. **Inline-кнопка** — кнопка под сообщением бота (`web_app` тип)
3. **Deep link** — `https://t.me/MyrmexBot?startapp=task_123` или `tg://resolve?domain=MyrmexBot&startapp=board_zavlab`
4. **Команда `/app`** — бот отправляет сообщение с inline-кнопкой «Открыть канбан»

### 3.2. Стартовый экран

При открытии TWA без параметров:
- Загружается последняя открытая доска (из `localStorage` или дефолт — ЗАВЛАБ)
- Если есть непрочитанные уведомления — показать бейдж на соответствующей доске
- Если есть просроченные задачи — показать toast «3 задачи просрочены»

### 3.3. Переключение досок

В TWA переключение между тремя досками (ЗАВЛАБ, МУРАВЕЙ, КОТ) реализуется через:

**Вариант A: Верхние табы (предпочтительный)**
```
┌─────────────────────────────────┐
│ 🔬 ЗАВЛАБ │ 🐜 МУРАВЕЙ │ 🐱 КОТ │  ← табы в хедере
├─────────────────────────────────┤
```

- Табы всегда видны, переключение мгновенное
- Активный таб подсвечен цветом доски (emerald/amber/sky)
- Haptic: `selectionChanged` при переключении
- Свайп по зоне контента переключает доски (как ViewPager)

**Вариант B: Боковая навигация (для широких экранов)**
- Sidebar с тремя досками + иконками
- На мобильных — сворачивается в гамбургер

### 3.4. Навигация по колонкам

На мобильных в TWA — одна колонка + горизонтальные табы (см. [мобильный дизайн](kanban-mobile.md)):
- Табы колонок: `[Бэклог] [К делу] [В работе] [Ревью] [Готово]`
- Свайп влево/вправо переключает колонки
- Индикатор `2/5` показывает текущую позицию
- Haptic: `selectionChanged` при переключении, `impact('light')` при достижении края

### 3.5. Навигация к карточке

**Из списка (доска):**
- Тап на карточку → Bottom Sheet с деталями
- BackButton закрывает Bottom Sheet

**Из deep link:**
- `tg://resolve?domain=MyrmexBot&startapp=task_T-123`
- TWA открывается → парсинг `startapp` → загрузка задачи → открытие Bottom Sheet

**Из уведомления бота:**
- Бот отправляет сообщение с inline-кнопкой «Открыть задачу»
- Кнопка содержит `web_app` с URL, включающим `?task=T-123`

### 3.6. Кнопка «Назад» (BackButton)

BackButton в TWA — системная кнопка «←» в шапке Telegram. Используется для:

| Состояние | Действие BackButton |
|-----------|---------------------|
| Bottom Sheet открыт | Закрыть Sheet |
| Форма создания/редактирования | Закрыть форму (с подтверждением при несохранённых данных) |
| Выбор колонки | Вернуться к карточке |
| Главный экран доски | Скрыть TWA (возврат в чат) |

**Правило:** BackButton всегда должен вести себя предсказуемо. Если есть несохранённые данные — `WebApp.showConfirm('Отменить изменения?')` перед закрытием.

## 4. Быстрые действия через ботов

### 4.1. Бот МУРАВЕЙ — управление задачами агентов

Бот МУРАВЕЙ — основной канал для работы агентов с канбан-доской. Команды:

#### Команды просмотра

| Команда | Действие | Ответ бота |
|---------|----------|------------|
| `/tasks` | Список активных задач агента | Inline keyboard с карточками |
| `/board` | Состояние доски МУРАВЕЙ | Компактный текст + inline-кнопка «Открыть TWA» |
| `/task T-123` | Детали задачи | Карточка задачи + inline-кнопки действий |
| `/blocked` | Заблокированные задачи | Список с причинами блокировки |
| `/today` | Задачи на сегодня | Фильтрованный список |

#### Команды действий

| Команда | Действие | Ответ бота |
|---------|----------|------------|
| `/done T-123` | Отметить задачу как выполненную | ✅ + обновлённая карточка |
| `/move T-123 review` | Переместить задачу в колонку | ✅ + подтверждение |
| `/block T-123 причина` | Заблокировать задачу | 🔒 + уведомление менеджеру |
| `/unblock T-123` | Снять блокировку | 🔓 + уведомление |
| `/comment T-123 текст` | Добавить комментарий | 💬 + обновлённая карточка |
| `/assign T-123 @agent` | Назначить задачу агенту | 👤 + уведомление агенту |

#### Команды создания

| Команда | Действие | Ответ бота |
|---------|----------|------------|
| `/new Название задачи` | Быстрое создание задачи | ✅ + ссылка на карточку |
| `/new Название \| описание \| приоритет` | Создание с параметрами | ✅ + карточка |
| `/newfrom` | Интерактивное создание (диалог) | Пошаговые вопросы |

### 4.2. Бот КОТ — личные задачи Евгения

Бот КОТ — персональный канал. Команды:

| Команда | Действие | Ответ бота |
|---------|----------|------------|
| `/my` | Мои активные задачи | Inline keyboard |
| `/add Название` | Быстрое добавление | ✅ + карточка |
| `/done ID` | Выполнить задачу | ✅ + прогресс дня |
| `/plan` | План на сегодня | Список + inline-кнопка «Открыть TWA» |
| `/week` | Обзор недели | Текст + статистика |
| `/idea текст` | Добавить идею в бэклог | 💡 + подтверждение |

### 4.3. Интерактивное создание задачи (диалог с ботом)

Когда агент вызывает `/newfrom`, бот ведёт пошаговый диалог:

```
Бот: 📋 Создание новой задачи. Введите название:
Агент: Рефакторинг API middleware
Бот: ✅ Название: "Рефакторинг API middleware"
    Описание (или /skip):
Агент: Переписать middleware на новый стек
Бот: ✅ Описание добавлено
    Приоритет: /critical /high /medium /low
Агент: /high
Бот: ✅ Приоритет: 🔴 Высокий
    Теги (через пробел, или /skip):
Агент: backend api refactor
Бот: ✅ Теги: #backend #api #refactor
    Колонка: /backlog /todo /in_progress
Агент: /todo
Бот: ✅ Колонка: К делу

    📋 Задача готова к созданию:
    ─────────────────────────
    Название: Рефакторинг API middleware
    Описание: Переписать middleware на новый стек
    Приоритет: 🔴 Высокий
    Теги: #backend #api #refactor
    Колонка: К дело
    ─────────────────────────

    /confirm — создать
    /cancel — отменить
    /edit — изменить поле
```

**Детали:**
- Каждый шаг — отдельное сообщение бота + ожидание ответа
- Таймаут: 60 секунд на шаг, затем авто-отмена
- `/cancel` на любом шаге — отмена создания
- `/skip` — пропустить опциональное поле
- После `/confirm` — задача создаётся через API, бот отправляет карточку с inline-кнопкой «Открыть в TWA»

### 4.4. Быстрые действия из inline-клавиатуры

Каждое сообщение бота с карточкой задачи содержит inline-клавиатуру:

```
┌─────────────────────────────────────────┐
│ 📋 Рефакторинг API middleware           │
│ 🔴 Высокий | 🤖 Агент Кошка            │
│ 🏷️ backend, api, refactor               │
│ 📅 Дедлайн: 2026-05-15                 │
├─────────────────────────────────────────┤
│ [✅ Готово]  [➡️ Переместить]  [✏️ Ред.]│
├─────────────────────────────────────────┤
│ [🔗 Открыть в TWA]  [💬 Комментарий]    │
└─────────────────────────────────────────┘
```

**Кнопки действий:**
- `✅ Готово` — callback `done:{task_id}` → перемещает в DONE + уведомление
- `➡️ Переместить` — callback `move:{task_id}` → бот спрашивает целевую колонку
- `✏️ Редактировать` — callback `edit:{task_id}` → интерактивное редактирование
- `🔗 Открыть в TWA` — `web_app` кнопка с URL `https://myrmexcontrol.shtab-ai.ru/board?task={task_id}`
- `💬 Комментарий` — callback `comment:{task_id}` → бот ожидает текст комментария

## 5. Deep links на карточки

### 5.1. Формат deep links

```
# Открыть конкретную задачу
https://t.me/MyrmexBot?startapp=task_{task_id}
tg://resolve?domain=MyrmexBot&startapp=task_T-123

# Открыть доску
https://t.me/MyrmexBot?startapp=board_{board_id}
tg://resolve?domain=MyrmexBot&startapp=board_zavlab

# Открыть доску с фильтром
https://t.me/MyrmexBot?startapp=board_{board_id}_filter_{filter}
tg://resolve?domain=MyrmexBot&startapp=board_muravey_filter_blocked

# Открыть колонку
https://t.me/MyrmexBot?startapp=board_{board_id}_col_{column}
tg://resolve?domain=MyrmexBot&startapp=board_zavlab_col_in_progress

# Создать задачу (предзаполненная форма)
https://t.me/MyrmexBot?startapp=new_{board_id}
tg://resolve?domain=MyrmexBot&startapp=new_muravey
```

### 5.2. Параметры startapp

| Параметр | Формат | Пример | Действие |
|----------|--------|--------|----------|
| `task_{id}` | `task_T-123` | `task_T-456` | Открыть задачу в Bottom Sheet |
| `board_{id}` | `board_zavlab` | `board_muravey` | Открыть доску (дефолтная колонка) |
| `board_{id}_col_{col}` | `board_zavlab_col_todo` | `board_kot_col_doing` | Открыть доску на конкретной колонке |
| `board_{id}_filter_{f}` | `board_muravey_filter_blocked` | `board_zavlab_filter_overdue` | Открыть доску с фильтром |
| `new_{board}` | `new_muravey` | `new_kot` | Открыть форму создания на доске |
| `new_{board}_proj_{id}` | `new_zavlab_proj_p-12` | — | Создать задачу в проекте |

### 5.3. Обработка deep link в TWA

```
1. TWA открывается с параметром startapp
2. initTWA() парсит window.Telegram.WebApp.initDataUnsafe.start_param
3. useDeepLink() хук определяет тип параметра:
   - task_* → загрузить задачу → открыть Bottom Sheet
   - board_* → переключить доску → опционально колонку/фильтр
   - new_* → открыть форму создания
4. Если данные ещё не загружены — показать skeleton loader
5. После загрузки — навигация к целевому элементу
```

### 5.4. Генерация deep links

Deep links генерируются в следующих местах:

| Источник | Что генерирует | Куда вставляется |
|----------|---------------|------------------|
| Карточка задачи в TWA | `task_{id}` | Кнопка «Поделиться» → `WebApp.switchInlineQuery()` |
| Уведомление бота | `task_{id}` | Inline-кнопка «Открыть задачу» |
| Комментарий в чате | `task_{id}` | Автоматическая ссылка в тексте (формат: `T-123` → кликабельная) |
| Отчёт бота | `board_{id}_filter_{f}` | Кнопка «Открыть в TWA» в дайджесте |

### 5.5. Шеринг карточки в Telegram

Из TWA пользователь может поделиться карточкой задачи:

```
1. В Bottom Sheet карточки — кнопка «📤 Поделиться»
2. Генерируется deep link: https://t.me/MyrmexBot?startapp=task_T-123
3. WebApp.switchInlineQuery(link, ['users', 'groups', 'channels'])
4. Пользователь выбирает чат → отправляется сообщение с карточкой задачи
5. Сообщение содержит: текст карточки + inline-кнопку «Открыть в TWA»
```

## 6. Создание задач из Telegram

### 6.1. Сценарии создания

| Сценарий | Канал | Сложность |
|----------|-------|-----------|
| Быстрое создание из чата с ботом | `/new Название` | 1 сообщение |
| Создание с параметрами | `/new Название \| описание \| приоритет` | 1 сообщение |
| Интерактивное создание | `/newfrom` → диалог | 5-7 сообщений |
| Создание из пересланного сообщения | Переслать → бот предлагает | 2-3 сообщения |
| Создание из голосового | Голосовое → Whisper → задача | 1 сообщение |
| Создание из TWA (FAB) | Кнопка «+» → full-screen форма | UI-форма |
| Создание из TWA (MainButton) | MainButton «+ Новая задача» → форма | UI-форма |

### 6.2. Создание из пересланного сообщения

Когда пользователь пересылает сообщение боту:

```
Бот: 📨 Получено пересланное сообщение:
    ─────────────────────────
    "Нужно срочно починить баг с авторизацией.
    Пользователи не могут войти через Telegram."
    ─────────────────────────

    Создать задачу из этого сообщения?

    [✅ Создать задачу]  [❌ Отменить]
```

При нажатии «Создать задачу»:
- Текст сообщения → `description` задачи
- Из заголовка сообщения (первые 50 символов) → `title`
- Приоритет по умолчанию: `medium` (или `high` если есть слова «срочно», «баг», «критично»)
- Тег автоматически: `from-telegram`
- Бот запрашивает колонку и исполнителя

### 6.3. Создание из голосового сообщения

```
1. Пользователь отправляет голосовое боту
2. Бот скачивает файл через getFile API
3. Отправляет на Whisper API (whisper-1)
4. Полученный текст → анализ:
   - Первая фраза → title
   - Остальное → description
   - Ключевые слова → теги («баг» → #bug, «срочно» → #urgent)
5. Бот отправляет распознанный текст + предзаполненную карточку
6. Пользователь подтверждает или редактирует
```

### 6.4. Создание из текстового сообщения (NLP)

Когда пользователь отправляет боту обычный текст (не команду):

```
Пользователь: Нужно обновить документацию к API до пятницы

Бот: 📋 Распознана задача:
    ─────────────────────────
    Название: Обновить документацию к API
    Дедлайн: Пятница, 16 мая 2026
    Теги: #docs #api
    ─────────────────────────

    [✅ Создать]  [✏️ Редактировать]  [❌ Нет]
```

**NLP-правила распознавания:**

| Ключевые слова | Действие |
|----------------|----------|
| «срочно», «критично», «ASAP», «горит» | Приоритет → `high` или `critical` |
| «баг», «ошибка», «не работает», «сломано» | Тег → `#bug`, тип → `bug` |
| «документация», «README», «docs» | Тег → `#docs` |
| «до пятницы», «к понедельнику», «завтра» | Парсинг даты → `dueDate` |
| «для клиента», «для пользователей» | Тег → `#user-facing` |
| «рефакторинг», «оптимизация», «чистка» | Тег → `#tech-debt` |

## 9. Синхронизация состояния

### 9.1. Проблема

TWA и бот — два независимых клиента, работающих с одним API. Возможны конфликты:
- Агент переместил задачу в TWA → бот должен обновить inline-клавиатуру
- Агент выполнил задачу через бота → TWA должна обновить доску при следующем открытии

### 9.2. Решение: API как source of truth

```
┌──────────┐     ┌──────────┐
│   TWA    │     │  Бот     │
│ (React)  │     │ (Node.js)│
└────┬─────┘     └────┬─────┘
     │                │
     │   HTTP API     │
     │◄──────────────►│
     │                │
     └───────┬────────┘
             │
     ┌───────▼───────┐
     │  myrmex.json  │
     │  (source of   │
     │   truth)      │
     └───────────────┘
```

- Все изменения идут через REST API
- Бот и TWA не общаются напрямую
- Конфликты разрешаются на уровне API (timestamp-based last-write-wins)

### 9.3. Стратегия обновления

| Клиент | Стратегия | Интервал |
|--------|-----------|----------|
| TWA | Загрузка при открытии + ручной refresh | По событию |
| TWA | Фоновый polling (если TWA открыта > 30 сек) | 30 сек |
| Бот | Webhook от API при изменении | Мгновенно |
| Бот | Обновление inline-сообщений через editMessageReplyMarkup | По событию |

### 9.4. Обновление inline-сообщений

Когда задача изменена через TWA, API уведомляет бота через webhook:
```
POST /api/webhook/bot-update
{
  "type": "task_moved",
  "taskId": "T-123",
  "from": "todo",
  "to": "in_progress",
  "chatId": 123456789,
  "messageId": 42
}
```

Бот обновляет inline-клавиатуру сообщения через `editMessageReplyMarkup`.

### 9.5. Optimistic updates в TWA

Для мгновенного отклика в TWA:
1. Пользователь перетаскивает задачу → UI обновляется сразу (optimistic)
2. Запрос уходит на сервер
3. При успехе — подтверждение (haptic `notification('success')`)
4. При ошибке — откат UI + haptic `notification('error')` + toast «Ошибка сохранения»

## 10. Безопасность и авторизация

### 10.1. Авторизация в TWA

TWA использует данные пользователя из `initDataUnsafe`:
```
1. TWA открывается → initTWA() получает user data
2. Данные проверяются через validateTWAInitData (HMAC-SHA256)
3. Если валидно → POST /auth/twa-login { initData }
4. Сервер проверяет подпись через bot token
5. Если ок → создаётся session cookie
6. Дальше — обычная cookie-based аутентификация
```

### 10.2. Авторизация бота

Бот использует отдельный API-ключ:
```
Authorization: Bearer {BOT_API_KEY}
```

Каждый бот (МУРАВЕЙ, КОТ) имеет свой ключ. Ключи хранятся в `.env`:
```
MURAVEY_BOT_TOKEN=123456:ABC-DEF
KAT_BOT_TOKEN=789012:GHI-JKL
BOT_API_KEY=myrmex-internal-api-key-2026
```

### 10.3. Проверка прав доступа

| Действие | Кто может |
|----------|-----------|
| Просмотр доски ЗАВЛАБ | Все авторизованные |
| Редактирование задач ЗАВЛАБ | Евгений + назначенные агенты |
| Просмотр доски МУРАВЕЙ | Все авторизованные |
| Редактирование задач МУРАВЕЙ | Только назначенный агент + Евгений |
| Просмотр доски КОТ | Только Евгений |
| Редактирование задач КОТ | Только Евгений |
| Удаление задач | Евгений + создатель задачи |
| Изменение конфигурации доски | Только Евгений |

### 10.4. Защита от подделки deep links

Deep links содержат только ID ресурса — не содержат команд. Все действия через deep link — только навигация (открыть задачу, открыть доску). Действия (переместить, удалить) требуют отдельного API-запроса с проверкой прав.

## 11. Обработка ошибок

### 11.1. Сетевые ошибки в TWA

| Ситуация | Поведение | Haptic |
|----------|-----------|--------|
| Нет сети при загрузке | Показать offline-экран с кнопкой «Повторить» | `notification('error')` |
| Таймаут запроса | Toast «Сервер не отвечает» + кнопка «Повторить» | `notification('error')` |
| 401 (сессия истекла) | Перенаправить на /login | — |
| 403 (нет прав) | Toast «Нет доступа» | `notification('warning')` |
| 404 (не найдено) | Toast «Задача не найдена» + возврат к доске | — |
| 500 (серверная ошибка) | Toast «Ошибка сервера» + кнопка «Повторить» | `notification('error')` |

### 11.2. Ошибки бота

| Ситуация | Поведение |
|----------|-----------|
| Неизвестная команда | `❓ Неизвестная команда. Используйте /help` |
| Неверный формат | `⚠️ Неверный формат. Пример: /done T-123` |
| Задача не найдена | `❌ Задача T-999 не найдена` |
| Нет прав | `🚫 У вас нет прав для этого действия` |
| Сервер недоступен | `🔌 Сервер временно недоступен. Попробуйте позже.` |
| Таймаут диалога | `⏰ Время ожидания истекло. Начните заново: /newfrom` |

### 11.3. Конфликты синхронизации

| Ситуация | Поведение |
|----------|-----------|
| Задача изменена другим клиентом | В TWA: toast «Задача была изменена. Обновить?» + кнопка |
| Задача удалена другим клиентом | В TWA: закрыть Bottom Sheet + toast «Задача удалена» |
| Одновременное редактирование | Last-write-wins + уведомление проигравшему |
