# Канбан-доски Myrmex Control — Визуальная концепция

> **Версия:** 1.0 | **Дата:** 2026-05-11 | **Автор:** Visual Design Architect
>
> Концепция трёх канбан-досок для проекта Myrmex Control.
> Не код — архитектура и визуальное проектирование.

---

## 1. Общая архитектура трёх досок

```
┌─────────────────────────────────────────────────────────────────────┐
│  Myrmex Control — Канбан-система                                  │
├──────────────────┬──────────────────┬───────────────────────────────┤
│   🔬 ЗАВЛАБ     │   🐜 МУРАВЕЙ     │   🐱 КОТ                      │
│   (лаборатория)  │   (AI-агенты)    │   (личные Евгения)            │
├──────────────────┼──────────────────┼───────────────────────────────┤
│ Проекты и задачи│ Задачи агентов   │ Личные задачи и цели          │
│ команды ЗавЛаб  │ и их выполнение  │                               │
└──────────────────┴──────────────────┴───────────────────────────────┘
```

**Принцип:** три доски — три разных контекста. Каждая имеет свой набор колонок, свой визуальный стиль, свои WIP-лимиты. Переключение между досками — через табы на странице `/board` или через параметр `?view=zavlab|muravei|kot`.

**URL-схема:**
- `/board/zavlab` — доска ЗавЛаб (по умолчанию)
- `/board/muravei` — доска МУРАВЕЙ
- `/board/kot` — доска КОТ

## 2. Доска «ЗАВЛАБ» — Лаборатория

### 2.1. Колонки (6 штук)

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│  📥      │ │  📋      │ │  🔨      │ │  👁️      │ │  ✅      │ │  🗄️      │
│ ИДЕИ     │ │ К ДЕЛУ   │ │ В РАБОТЕ│ │ РЕВЬЮ    │ │ ГОТОВО   │ │ АРХИВ    │
│          │ │          │ │          │ │          │ │          │ │          │
│ ∞        │ │ WIP: 8   │ │ WIP: 3   │ │ WIP: 4   │ │ ∞        │ │ ∞        │
│          │ │          │ │          │ │          │ │          │ │          │
│ Серый    │ │ Синий    │ │ Янтарный │ │ Фиолетов.│ │ Зелёный  │ │ Тёмно-   │
│ #6b7280  │ │ #3b82f6  │ │ #f59e0b  │ │ #8b5cf6  │ │ #22c55e  │ │ серый    │
│          │ │          │ │          │ │          │ │          │ │ #374151  │
└──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

| # | Колонка | Цвет | WIP | Назначение |
|---|---------|------|-----|------------|
| 1 | **Идеи** | Серый `#6b7280` | ∞ | Несортированные идеи, сырые концепты, feature requests. Без ограничений — сюда можно кидать всё. |
| 2 | **К делу** | Синий `#3b82f6` | **8** | Отобранные, приоритизированные задачи. Готовы к работе. WIP 8 — чтобы не скапливалось 100 «важных» задач. |
| 3 | **В работе** | Янтарный `#f59e0b` | **3** | Задачи, над которыми кто-то активно работает. WIP 3 — фокус на завершении, а не на распылении. |
| 4 | **Ревью** | Фиолетовый `#8b5cf6` | **4** | Задачи на проверке / code review / тестировании. WIP 4 — буфер между «сделал» и «принял». |
| 5 | **Готово** | Зелёный `#22c55e` | ∞ | Завершённые задачи. Сюда попадают после подтверждения. |
| 6 | **Архив** | Тёмно-серый `#374151` | ∞ | Отменённые, дубликаты, неактуальные. Отдельная колонка вместо удаления — для истории. |

**Почему 6, а не 5:** Текущая доска имеет 5 колонок (backlog/todo/in_progress/review/done). Добавляем «Архив» для отменённых задач вместо их удаления. Это даёт полную картину и позволяет анализировать причины отмен.

### 2.2. Визуальный стиль

- **Заголовок доски:** `🔬 ЗАВЛАБ` + счётчик задач (напр. `23 задачи`)
- **Фон колонок:** `bg-card` (`#1e293b`) с тонкой границей `border-border`
- **Акцентный цвет доски:** emerald `#10b981` (полоска сверху заголовка, активный таб)
- **Иконка доски:** `FlaskConical` (Lucide) или кастомная SVG-колба
- **Header bar:** тёмный `bg-background`, с поиском по задачам и фильтрами (по проекту / приоритету / тегам)

### 2.3. Swimlanes (горизонтальные дорожки)

**Нужны.** Группировка по проектам.

```
┌─────────────────────────────────────────────────────────────────────┐
│  📥 ИДЕИ (∞)                                                        │
├─────────────────────────────────────────────────────────────────────┤
│  ▸ Проект: Myrmex Control                              [3 задачи]  │
│    ┌─────────┐ ┌─────────┐ ┌─────────┐                             │
│    │ Карточка│ │ Карточка│ │ Карточка│                             │
│    └─────────┘ └─────────┘ └─────────┘                             │
├─────────────────────────────────────────────────────────────────────┤
│  ▾ Проект: VPN Infrastructure                          [7 задач]  │
│    ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│    │ Карточка│ │ Карточка│ │ Карточка│ │ Карточка│ │ Карточка│   │
│    └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘   │
├─────────────────────────────────────────────────────────────────────┤
│  ▸ Проект: LabDoctorM (archived)                        [0 задач]  │
└─────────────────────────────────────────────────────────────────────┘
```

**Правила:**
- Swimlanes сворачиваются/разворачиваются кликом на имя проекта
- Свёрнутый проект показывает только счётчик задач
- Архивные проекты свёрнуты по умолчанию
- Цвет полоски swimlane = цвет проекта (`project.color`)

**Когда отключать swimlanes:** если задач < 10 или все задачи одного проекта — переключение в плоский вид (toggle в header).

│  💤      │ │  🎯      │ │  ⚡      │ │  🔍      │ │  ✅      │
│ ОЖИДАНИЕ │ │ НАЗНАЧЕНО│ │ ВЫПОЛНЯЕТ│ │ ПРОВЕРКА │ │ СДАНО    │
│          │ │          │ │          │ │          │ │          │
│ ∞        │ │ WIP: 10  │ │ WIP: 5   │ │ WIP: 5   │ │ ∞        │
│          │ │          │ │          │ │          │ │          │
│ Серый    │ │ Голубой  │ │ Янтарный │ │ Фиолетов.│ │ Зелёный  │
│ #6b7280  │ │ #0ea5e9  │ │ #f59e0b  │ │ #8b5cf6  │ │ #22c55e  │

| # | Колонка | Цвет | WIP | Назначение |
|---|---------|------|-----|------------|
| 1 | **Ожидание** | Серый `#6b7280` | ∞ | Задачи, ожидающие назначения агенту. Агенты не назначены. |
| 2 | **Назначено** | Голубой `#0ea5e9` | **10** | Задача назначена агенту, но он ещё не начал. Очередь на выполнение. |
| 3 | **Выполняет** | Янтарный `#f59e0b` | **5** | Агент активно работает. WIP 5 — максимум 5 параллельных задач для всей колонии. |

### 3.2. Визуальный стиль

- **Заголовок доски:** `🐜 МУРАВЕЙ` + статус колонии (напр. `3/5 агентов активны`)
- **Акцентный цвет доски:** amber `#f59e0b` (полоска сверху заголовка)
- **Иконка доски:** `Bot` (Lucide) или кастомная SVG-муравей
- **Header bar:** фильтр по агенту, по статусу агента, по приоритету задачи

### 3.3. Swimlanes

**Нужны.** Группировка по агентам.

```
┌─────────────────────────────────────────────────────────────────────┐
│  ⚡ ВЫПОЛНЯЕТ (WIP: 2/5)                                           │
├─────────────────────────────────────────────────────────────────────┤
│  ▾ 🐜 OWL (admin) — статус: working                    [2 задачи]  │
│    ┌─────────────────────┐ ┌─────────────────────┐                 │
│    │ 🔴 Критический      │ │ 🟡 Высокий           │                │
│    │ Аудит безопасности  │ │ Рефакторинг Board   │                 │
│    │ ⏱️ 2ч 15м           │ │ ⏱️ 45м              │                │
│    └─────────────────────┘ └─────────────────────┘                 │
├─────────────────────────────────────────────────────────────────────┤
│  ▸ 🐜 Mur (operator) — статус: idle                     [0 задач]  │
├─────────────────────────────────────────────────────────────────────┤
│  ▾ 🐜 ZavLab (operator) — статус: working               [1 задача] │
│    ┌─────────────────────┐                                         │
│    │ 🟢 Средний           │                                        │
│    │ Обновить документацию│                                        │
│    │ ⏱️ 1ч 30м           │                                        │
│    └─────────────────────┘                                         │
└─────────────────────────────────────────────────────────────────────┘
```

**Правила:**
- Каждый swimlane = один агент
- Имя агента + роль + статус (цветной индикатор: зелёный=working, серый=idle, красный=error, тёмный=offline)
- Свёрнутые агенты (idle/offline) группируются внизу
- WIP-лимит на агента: **максимум 2 задачи в колонке «Выполняет»** на одного агента

## 4. Доска «КОТ» — Личные задачи Евгения

### 4.1. Колонки (4 штуки)

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│  📥      │ │  ⭐      │ │  🔨      │ │  ✅      │
│ ВХОДЯЩИЕ │ │ СЕГОДНЯ  │ │ В РАБОТЕ│ │ ГОТОВО   │
│          │ │          │ │          │ │          │
│ ∞        │ │ WIP: 5   │ │ WIP: 2   │ │ ∞        │
│          │ │          │ │          │ │          │
│ Серый    │ │ Зелёный  │ │ Янтарный │ │ Синий    │
│ #6b7280  │ │ #10b981  │ │ #f59e0b  │ │ #3b82f6  │
└──────────┘ └──────────┘ └──────────┘ └──────────┘
```

| # | Колонка | Цвет | WIP | Назначение |
|---|---------|------|-----|------------|
| 1 | **Входящие** | Серый `#6b7280` | ∞ | Все новые задачи сюда. Мозговой дамп. Без ограничений. |
| 2 | **Сегодня** | Изумрудный `#10b981` | **5** | Задачи, запланированные на сегодня. WIP 5 — реалистичный лимит. |
| 3 | **В работе** | Янтарный `#f59e0b` | **2** | Максимум 2 задачи одновременно. Фокус. |
| 4 | **Готово** | Синий `#3b82f6` | ∞ | Завершённые. Ежедневно очищается (переносится в архив). |

**Ключевое отличие:** минимализм. 4 колонки вместо 6. Нет «Ревью» — Евгений сам принимает работу. Нет «Архива» — завершённые задачи сбрасываются раз в день.

### 4.2. Визуальный стиль

- **Заголовок доски:** `🐱 КОТ` + дата (напр., `Понедельник, 11 мая`)
- **Акцентный цвет доски:** emerald `#10b981` (полоска сверху)
- **Иконка доски:** `User` (Lucide) или кастомная SVG-кот
- **Header bar:** дата, прогресс-бар дня (3/5 выполнено), кнопка «Сбросить готово»

### 4.3. Swimlanes

**Не нужны.** Один человек — один контекст. Вместо этого — **теги-фильтры** сверху:

```
[Все] [💼 Работа] [🏠 Дом] [💪 Здоровье] [📚 Обучение] [🔧 Администрирование]
```

## 5. Карточка задачи — внутренняя структура

### 5.1. Компактный вид (на доске)

```
┌─────────────────────────────────────────────┐
│ 🔴 Заголовок задачи                         │  ← Приоритет-точка + заголовок
│                                             │
│ 📝 Краткое описание в одну-две строки...   │  ← Описание (опционально)
│                                             │
│ ┌─────┐ ┌─────┐ ┌─────┐                    │  ← Теги (макс 3, остальные +N)
│ │ tag │ │ tag │ │ +2  │                    │
│ └─────┘ └─────┘ └─────┘                    │
│                                             │
│ ████████░░░░░░░░ 60%                        │  ← Прогресс-бар (если есть чеклист)
│                                             │
│ 🐜 OWL    ⏱️ 2ч    📅 11 мая               │  ← Агент + время + дата
└─────────────────────────────────────────────┘
```

### 5.2. Элементы карточки (детально)

| Элемент | Видимость | Описание |
|---------|-----------|----------|
| **Приоритет** | Всегда | Цветная точка слева (🔴 critical, 🟠 high, 🟡 medium, ⚪ low) + левая граница карточки того же цвета |
| **Заголовок** | Всегда | Жирный, 1 строка, `truncate`. Клик → открыть детали |
| **Описание** | Если есть | `line-clamp-2`, мелкий шрифт, `text-muted-foreground` |
| **Теги** | Если есть | Макс 3 видимых, остальные как `+N`. Цвета из палитры тегов |
| **Прогресс** | Если есть чеклист | Тонкий прогрес-бар `4px`, emerald для выполненных |
| **Агент** | Если назначен | Аватар/иконка агента (первая буква имени в цветном круге) |
| **Время** | Если в работе | `⏱️ Xч Ym` — время с момента перемещения в «В работе» |
| **Дата** | Всегда | `📅 DD MMM` — дата создания или дедлайн |
| **Чеклист-счётчик** | Если есть | `☑️ 3/5` — выполнено / всего |
| **Старение** | Авто | Фон карточки меняется при застревании (см. раздел 7) |
| **Зависимости** | Иконка | `🔗 2` — есть зависимости (блокирует/заблокирована) |

## 6. Вложенность — эпики и под-задачи

### 6.1. Иерархия

```
Эпик (Epic)
├── Задача 1 (Task)
│   ├── Под-задача 1.1 (Subtask)
│   └── Под-задача 1.2 (Subtask)
├── Задача 2 (Task)
└── Задача 3 (Task)
```

### 6.2. Визуализация на доске

**Эпик** — карточка с визуальным отличием:

```
┌─────────────────────────────────────────────┐
│ 📦 ЭПИК: Рефакторинг канбан-доски           │  ← Иконка 📦 + метка «ЭПИК»
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│ ██████████████░░░░░░░░░░ 60%                │  ← Прогресс по дочерним задачам
│ ☑️ 3/5 задач выполнено                      │  ← Счётчик дочерних
│ ┌──────┐ ┌──────┐                           │
│ │react │ │ types│ ← Теги                    │
│ └──────┘ └──────┘                           │
└─────────────────────────────────────────────┘
```

**Отличия эпика от обычной карточки:**
- Иконка `📦` (или `Layers` из Lucide) перед заголовком
- Метка `ЭПИК` мелким шрифтом
- Прогресс-бар агрегирует дочерние задачи
- Счётчик `☑️ X/Y` — выполненные / все дочерние
- Клик → раскрывает список дочерних задач (inline expand или slide-over)

**Под-задача** — отображается только внутри эпика, не на основной доске. На доске виден только эпик.

### 6.3. Карточка → под-доска

**Не реализовывать на первом этапе.** Вместо этого:
- Клик по карточке → **slide-over панель** справа с деталями
- В slide-over: описание, чеклист, комментарии, история перемещений
- Для эпиков: в slide-over — список дочерних задач с чекбоксами

## 7. Старение карточек — визуализация застревания

### 7.1. Логика старения

| Возраст в колонке | Визуальный эффект | Уровень |
|---|---|---|
| < 24 часов | Без изменений | Норма |
| 1-3 дня | Лёгкое пожелтение фона | Внимание |
| 3-5 дней | Янтарная тень + иконка `⏰` | Тревога |
| 5-7 дней | Красная пульсирующая граница | Критично |
| > 7 дней | Красная заливка + иконка `🚨` + авто-эскалация | Блокер |

### 7.2. Визуальные состояния

```
НОРМА (0-1 день):          ВНИМАНИЕ (1-3 дня):
┌──────────────────────┐   ┌──────────────────────┐
│ Обычная карточка     │   │ Обычная карточка     │
│ bg: background       │   │ bg: amber-50         │
│ border: border       │   │ border: amber-200    │
└──────────────────────┘   └──────────────────────┘

ТРЕВОГА (3-5 дней):        КРИТИЧНО (5-7 дней):
┌─ ⏰ ─────────────────┐   ┌─ 🚨 ─────────────────┐
│ Карточка             │   │ Карточка             │
│ bg: amber-100        │   │ bg: red-50           │
│ border: amber-400    │   │ border: red-400      │
│ shadow: amber glow   │   │ animation: pulse     │
└──────────────────────┘   └──────────────────────┘

БЛОКЕР (>7 дней):
┌─ 🚨 ─────────────────┐
│ Карточка             │
│ bg: red-100          │
│ border: red-500      │
│ ⚠️ ЗАСТРЛА 7+ ДНЕЙ  │
│ [Разблокировать]     │
└──────────────────────┘
```

### 7.3. Исключения из старения

- Колонка «Идеи» / «Входящие» — **старение отключено** (там всё лежит долго по дизайну)
- Колонка «Готово» / «Сдано» — **старение отключено** (задача завершена)
- Колонка «Архив» — **старение отключено**
- Задача с тегом `#waiting` / `#blocked` — **старение приостановлено**, вместо этого показывается иконка `⏸️ Ожидание`

### 7.4. Агрегация в заголовке колонки

В заголовке каждой колонки показывается не только количество задач, но и «здоровье»:

```
┌──────────────────────────────────┐
│ 🔨 В работе          3 ⚠️       │  ← 3 задачи, 1 из них старая
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
```

Иконки в заголовке:
- `✅` — все задачи свежие
- `⚠️` — есть задачи старше 3 дней
- `🚨` — есть задачи старше 5 дней

## 11. Мобильная адаптация

### 11.1. Горизонтальный скролл колонок

На мобильных (< 768px) — колонки скроллятся горизонтально, одна за другой:

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 📥 Идеи  │→ │ 📋 К делу│→ │ 🔨 В раб.│  → ...
│          │  │          │  │          │
│ [карточ.]│  │ [карточ.]│  │ [карточ.]│
│ [карточ.]│  │          │  │ [карточ.]│
└──────────┘  └──────────┘  └──────────┘
```

- Видна 1 колонка + края следующей (намёк на скролл)
- Свайп влево/вправо для навигации
- Заголовок колонки закреплён (sticky)

### 11.2. Drag-and-drop на мобильных

- Долгий тап (long press) → карточка «поднимается» → перетаскивание
- Альтернатива: тап по карточке → slide-over с кнопками «Переместить в...»
- Вибрация (haptic feedback) при поднятии и размещении

### 11.3. Swimlanes на мобильных

- Сворачиваются автоматически
- Показывается только первая swimlane, остальные — по свайпу вверх

## 15. Принципы дизайна

1. **Прогрессивное раскрытие:** на доске — только самое важное. Детали — в slide-over. Никаких переходов на другую страницу.

2. **Цвет как сигнал:** каждый цвет имеет семантику. Красный = проблема/критический. Янтарный = внимание/в работе. Зелёный = готово/успех. Серый = нейтральный/ожидание.

3. **WIP спасает от хаоса:** жёсткие лимиты на «В работе» заставляют завершать, а не начинать.

4. **Старение — это не баг, это фича:** визуализация застревания — главный инструмент для выявления блокеров.

5. **Три доски — три контекста:** не смешивать. Задача агента != задача проекта != личная задача.

6. **Мобильный первый:** горизонтальный скролл, long-press для DnD, sticky заголовки.

7. **Empty state — это не пустота, это приглашение:** каждая пустая доска/колонка подсказывает что делать.
